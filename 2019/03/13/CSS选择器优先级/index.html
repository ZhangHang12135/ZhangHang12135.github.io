CSS选择器优先级

发表于 2019-03-13 更新于 2021-12-19 分类于 css

优先级的计算

优先级就是分配给指定的css声明的一个权重，它由匹配的选择器中的每一种选择器类型的数值计算的。
而当声明的优先级相等的时候，css最后的那个声明会被应用到元素上。

当同一个元素有多个声明的时候，优先级才会有意义。因为每一个直接作用于元素的CSS规则总是会接管/覆盖（take over）该元素从祖先元素继承而来的规则。

选择器的种类

严格来讲，css的选择器分为3种：

标签名选择器(h1)
类选择器(.class)
ID选择器(#id)

而所谓的群组选择器和后代选择器不过是对前三种的扩展应用。但是一般情况下会细分为如下几种:

!important 权值最大 1.0.0.0
ID选择器(#id) 0.1.0.0
类选择器(.class) 0.0.1.0
标签选择器(h1) 0.0.0.1
通配符选择器(*) 0.0.0.0

严格来说!important并不能算作选择器，但是它的优先级是凌驾于所有的，所以就单独提出来了
还有一个就是内联的样式引用style{…}，这是仅次于!important的一个
这里的话，规则大概是这个样子的，权值相加大的显示，同级的就是后来的厉害
所以一般就是这样的一个排名(敲黑板)

!important > 内联 > ID选择器 > 类选择器 | 属性选择([attribute]) > 标签选择器 > 通配符 > 浏览器默认

CSS的使用建议

所谓高效就是让浏览器查找更少的元素标签来确定匹配的style元素。
1.不要再ID选择器前使用标签名
解释：ID选择是唯一的，加上标签名相当于画蛇添足了，没必要。
2.不要在类选择器前使用标签名
解释：如果没有相同的名字出现就是没必要，但是如果存在多个相同名字的类选择器则有必要添加标签名防止混淆如（p.colclass{color：red;} 和 span.colclass{color:red;}
3.尽量少使用层级关系；#divclass p.colclass{color:red;}改为 .colclass{color:red;}
